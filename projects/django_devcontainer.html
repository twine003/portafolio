Project Overview

Developing Django web apps with a Postgres database can be slow and complex due to environment configuration. Docker and devcontainers streamline setup so you can build features faster.

The Problem

TWhen starting a new Django project, I wanted an easy way to get up and running with Django and Postgres. However, most tutorials or sample projects skip over the details of configuring the development environment. There is a lot of boilerplate code needed to link Django to the database, run migrations, connect Django’s runserver to the Docker containers etc.

I could not find a simple, bare-bones template to clone and start developing immediately with Django and Postgres in Docker. So I created this project to provide a starter devcontainer config that takes care of the initial setup hassles.
